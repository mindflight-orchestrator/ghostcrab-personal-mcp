# OpenClaw App Patterns

Start from:

- [shared/APP_PATTERNS.md](../../shared/APP_PATTERNS.md)

OpenClaw-specific additions:

- keep packs small and operational
- surface blockers immediately
- use recurring checklists for long-running agents
- prefer explicit escalation language when gaps remain unresolved
- keep first-turn GhostCrab onboarding intake-only when the request is still fuzzy
- end meaningful sessions and phase boundaries with checkpoints
- preserve transition rationale before overwriting current-state records when recovery would otherwise lose why the state changed

# Posts

Use this folder for article-style documentation that can be published through the GitHub wiki or linked from release notes.

Suggested front matter for future posts:

```markdown
---
title: "Article title"
date: "YYYY-MM-DD"
tags: ["ghostcrab", "mindbrain"]
---
```

Keep evergreen product docs in the main documentation folders, and use posts for narrative walkthroughs, announcements, comparisons, and implementation notes.
